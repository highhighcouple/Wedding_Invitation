# 개발자 가이드 및 구현 기획서 (Developer.md)

## 1. 프로젝트 개요
**High Quality Mobile Wedding Invitation**
전 세계 어디서나 접근 가능한 반응형 모바일 청첩장 사이트입니다. 아름다운 디자인과 인터랙티브한 요소, 그리고 강력한 관리자 기능을 제공합니다. Github Pages를 통해 호스팅되며, 백엔드 로직 없이 순수 HTML/CSS/JS로 동작하도록 설계되었습니다 (데이터는 로컬 스토리지 또는 별도 파일 기반 관리 시뮬레이션).

## 2. 기술 스택 및 제약 사항
- **Frontend**: HTML5, CSS3 (Vanilla), JavaScript (Vanilla)
- **Hosting**: GitHub Pages (Static hosting only)
- **Design System**:
    - **반응형**: 4K 대형 모니터부터 모바일(320px~)까지 유연하게 대응
    - **크로스 브라우징**: Chrome, Safari, Firefox, Edge, Opera 등 주요 브라우저 지원
    - **타이포그래피**: Google Fonts (다국어 지원 무료 폰트 활용)
- **Data Management**: `admin.html`을 통한 JSON/Local Storage 기반 데이터 관리 (서버리스 환경 고려)

## 3. 주요 기능 명세

### 3.1. 공통 기능
- **다국어 지원 (i18n)**:
    - 접속 IP 기반 국가 자동 감지 및 언어 설정 (한국어, 영어, 일본어, 중국어(간/번), 프랑스어, 독일어, 이탈리아어, 스페인어 등)
    - 상단 메뉴바에서 수동 언어 변경 가능
    - 구글 번역 API 연동 (자동 번역 지원)
- **TTS (Text-to-Speech)**: 사이트 내용을 여성 음성으로 읽어주는 기능 리더기
- **배경 음악 (BGM)**: 업로드된 음원 자동 반복 재생 (재생 컨트롤러 제공)
- **인터랙티브 효과**:
    - 스크롤 시 섹션 페이드 인/아웃
    - 터치/클릭 시 파티클 이펙트 (별가루 등)
    - 배경 애니메이션 (꽃잎, 눈송이, 낙엽 등 선택 가능)

### 3.2. 사용자 페이지 (`index.html`)
1.  **인트로**: 딤드 레이어 및 스크롤 유도 애니메이션
2.  **메인 섹션**: 상단 이미지, 오버레이 텍스트, 초대 문구
3.  **신랑/신부 정보**: 이름, 부모님 성함, 서열(장남/장녀 등), 전화번호 연동
4.  **달력 & D-Day**: 결혼식 날짜 디스플레이, 디데이 카운터
5.  **갤러리**:
    - 메인 슬라이드 (자동 전환)
    - 그리드 뷰 (터치 시 확대 팝업)
6.  **마음 전하실 곳 (계좌)**: 신랑/신부 측 계좌번호 복사 기능
7.  **오시는 길**:
    - 주소 정보, 카카오맵/구글맵/티맵/네이버지도 연동
    - 지도 이미지 다운로드 기능
    - 교통편(버스/지하철/셔틀) 안내 텍스트
8.  **방명록**: 작성 및 조회 (페이지네이션)
9.  **참석 여부 (RSVP)**: 참석/불참/식사여부 제출 폼
10. **SNS 공유**: 카카오톡, 페이스북, 인스타그램, X, 라인, 텔레그램 공유 및 링크 복사

### 3.3. 관리자 페이지 (`admin.html`)
- **접근 보안**: 비밀번호 인증 (초기: `admin1234!`)
- **데이터 관리**:
    - 이미지/텍스트 업로드 및 수정
    - 섹션 순서 변경 (드래그 앤 드롭)
    - 섹션 노출 여부 설정 (접기/펼치기)
    - 배경 효과 및 BGM 설정
    - 방명록 관리 (삭제/차단)
    - 참석 여부 리스트 확인 및 CSV 다운로드
    - IP 로그 확인

## 4. 디자인 가이드
- **컨셉**: 순백의 고급스럽고 고풍스러운 웨딩 테마
- **인터랙션**: 부드러운 모션, 은은한 발광 효과, 고급스러운 트랜지션
- **레이아웃**: 모바일 퍼스트 뷰, 정보의 가독성 최우선

## 5. 보안 및 설정 (`secrets.toml`)
- 외부 API 키 및 관리자 비밀번호 관리 (GitHub Secrets와 연동 고려하되, 클라이언트 사이드 노출 최소화 방안 필요)
- 키 목록: GitHub Access Token, Admin Password, Google Maps API, Kakao JS Key 등

## 6. 구현 로드맵
1.  기본 HTML 구조 및 CSS 스타일링 (Design System 구축)
2.  `admin.html` 기능 구현 (데이터 구조 설계)
3.  `index.html` 데이터 바인딩 및 렌더링 로직 구현
4.  외부 API 연동 (지도, 공유, 번역)
5.  인터랙티브 효과 및 애니메이션 적용
6.  최종 테스트 및 배포
